Feature Flow:
1. User selects member from MembersListPage
2. Navigate to MemberDetailPage showing member info and courses
3. MemberDetailPage loads enrolled courses and available courses via API
4. User clicks "Join" button on available course
5. API validates member/course existence and prevents duplicates
6. Creates enrollment record in Firestore subcollection
7. UI updates to show course in enrolled section
8. Success message displayed to user

Validation & Error Handling:
✅ Member ID and Course ID validation
✅ Duplicate enrollment prevention  
✅ Member/Course existence checks
✅ User-friendly error messages
✅ Loading states and visual feedback
